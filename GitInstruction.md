# Что такое Git

**Git** - это одна из реализаций распределенных систем контроля версий, имеющая как локальные, так и удаленные репозитрии. Является самой опулярной реализацией систем контроля версий в мире.

Главное преимущество git заключается в том, что он может хранить огромное количество версий и изменений проекта потребляя мало памяти, т.к. в каждой версии сохраняются только внесенные изменения, а не весь проект.

Кроме того, git позволяет настроить и поддерживать структуру проекта с большим количеством одновременно работающих участников.

# I. Подготовка репозитория

Для создания репозитория необходимо выполнить команду ***git init*** в папке с репозиторием.

Обязательно проверяем папку командой ***git status*** до и после, чтобы убедиться, что исходная папка не была репозиторием, и что репозиторий успешно создан.

P.S. Возможно вначале придется ввести имя и эл.почту пользователя командами 

***git config --global user.name имя_пользователя*** и

 ***git config --global user.mail электронная_почта***

# II. Работа с коммитами

## 1. Git add

Для добавления файлов с сохраненными изменениями в комит используется команда ***git add "имя_файла"***

Не забываем сохраняться перед коммитами, или можно включить автосохранение в настройках.

## 2. Создание коммитов

Для того, чтобы создать коммит необходимо выполнить команду ***git commit -m "сообщение"***

Каждый коммит встраивается в структуру проекта с соблюдением определенной последовательности и получает свой уникальный хэш-код.

Для просмотра разницы между текущим сохраненным файлом и последним коммитом используется команда ***git diff***, если после нее добавить два хэша коммитов, то можно сравнивать коммиты между собой.

Для проверки незакоммиченных изменений файла можно использовать команду ***git status***

## 3. Просмотр коммитов

Для того, чтобы просмотреть все доступные в данной ветке коммиты, нужно ввести команду *git log* или команду ***git log --graph*** для визуализации связей между коммитами


# III. Работа с ветками

## 1. Создание ветки

Для того, чтобы создать ветку, нужно использовать ***git branch имя_ветки*** или ***git checkout -b "имя ветки"***

Большое значение имеет на какой ветке находишься, когда создаешь другую ветку, несколько веток могут либо последовательно выходить друг из друга, или параллельно из одной ветки.

Набор доступных коммитов меняется в зависимости от выбранной ветки. некоторые коммиты могут не входить в выбранную ветку и будут не видны, если их не вызвать специально.

## 2. Просмотр веток

Для того, чтобы увидеть все ветки проекта, нужно ввести команду ***git branch***

## 3. Удаление ветки

Для того, чтобы удалить ненужную ветку можно использовать команду ***git branch -d "имя_ветки"***

## 4. Слияние веток

Для того, чтобы слить ветки, нужно перейти в базовую ветку и применить команду ***git merge "имя_ветки"*** с указанием ветки, которую мы хотим влить в базовую ветку.

## 5. Переход между ветками

Для того, чтобы перейти между ветками проекта, нужно ввести команду ***git checkout "branch_name"***

# IV. Работа с удаленными репозиториями

Удаленные репозитории создаются на соответствующих сервисах в локальных сетях или сети Интернет. Наиболее популярным сервисом удаленных репозиториев является **GitHub** 

## 1. Создание удаленного репозитория

Для создания удаленного репозитория достаточно зарегистрировать аккаунт на сервисе GitHub и создать новый репозиторий нажав соответствующую кнопку и проставив настройки, в частности публичный или приватный доступ к репозиторию.

## 2. Закачка локального репозитория на GitHub

Для того, чтобы отправить свой локальный репозиторий на GitHub, нужно создать пустой репозиторий на GitHub,  скопировать ссылку на него под иконкой ***Code*** и ввести команды в терминале git

***git remote add origin ссылка_на_удаленный_репозиторий*** 

этим мы устанавливаем связь локального и удаленного репозитория

***git branch -m main***

эта команда задает локальную ветку master как main в соответствии с принятым наименованием на GitHub

***git push -u origin main***

Эта команда пушит нашу ветку main из локального репозитория в привязанный удаленный репозиторий

Далее можно пользоваться командой ***git push***

Возможно при первом пуше понадобится авторизовать свой git на GitHub, затем можно будет беспрепятственно пушить свои репозитории на GitHub.

## 3. Клон репозитория на локальный компьютер

Для того, чтобы получить весь репозиторий из GitHub на локальный компьютер, достаточно скопировать ссылку на него под иконкой ***Code***, создать папку на локальном компьютере, открыть ее в git **не инициализируя**, и ввести ***git clone ссылка_на_репозиторий*** в терминале git. Полученный таким образом репозиторий не требует повторной инициализации. Не забыть перейти из своей папки в только что скачанный репозиторий командой ***cd имя_папки*** или вывзовом терминала из папки.

## 4. Выкачивание актуальной версии проекта

Для того, чтобы выкачать изменения из привязанного удаленного репозитория в локальный репозиторий используется команда ***git pull origin имя_ветки*** или просто ***git pull***. Эта команда помимо закачки изменений сразу делает их автоматическое слияние с локальным репозиторием, поэтому возможны конфликты. Для более аккуратного скачивания изменений следует использовать команду ***git fetch***, она просто скачает все недостающие коммиты в ветке без слияния с текущей версией файла.

## 5. Присоединение к чужому проекту 

Если интересующий нас репозиторий уже создан другим пользователем и размещен в открытом доступе, достаточно найти этот репозиторий и нажать иконку ***Fork***, это создаст копию репозитрия на вашем аккаунте, сохраняя связь с исходным репозиторием на аккаунте создателя проекта.

## 6. Работа в репозитории проекта

Для работы в форке репозитория уже созданного проекта, следует прочитать файл README, создать свою отдельную ветку для работы и вносить изменения только на ней. 

## 7. Отправка изменений с локального компьютера на GitHub

Для того, чтобы отправить сделанные в репозитории изменения в удаленный репозиторий на своем аккаунте в GitHub, нужно ввести команду ***git push --set-upstream origin имя_ветки***. Этим вы отправите свою рабочую ветку с локального компьютера на свой одноименный репозиторий на GitHub. Далее можно пользоваться командой ***git push***

Возможно при первом пуше понадобится авторизовать свой git на GitHub, затем можно будет беспрепятственно пушить свои репозитории на GitHub.

## 8. Pull request своих изменений владельцу проекта

После пуша своих изменений в форк удаленного репозитория на GitHub появится возможность отправки этих изменений автору проекта (создателю репозитория). Если у вас нет безусловных прав доступа к проекту для простого пуша, для этого понадобится нажать иконки ***Compare & Pull request*** или ***Open pull request***, добавить описание своих изменений в форме, которая последует за этим и отправить пулл реквест. Затем следует ждать пока этот реквест будет принят или отклонен владельцем проекта. 

# V. Вспомогательные команды

- Для роверки текущей версии git используется команда ***git --version***, желательно это делать сразу после установки для проверки работоспособности git. 

- Для того, чтобы очистить терминал, нужно ввести команду ***clear***

- Для возвращения в терминал из лога можно ввести ***q***

- Для создания папки можно ввести ***mkdir имя_папки***

- Для перехода между папками можно использовать команду ***cd имя_папки***

- Для вывода содержимого папки можно ввести команду ***ls***

- Для просмотра возможных аттрибутов команды можно ввести ***имя_команды -h*** или ***имя_команды --help***

- Для того, чтобы скрыть файлы, которым не требуется поддержка версионности, используется файл ***.gitignore*** в репозитории, в него пишутся игнорируемые git файлы, его нужно сохранить и закоммитить один раз.
