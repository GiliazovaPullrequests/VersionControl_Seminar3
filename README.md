# VersionControl_Seminar3
Репозиторий для пулл-реквестов по Введению в контроль версий в Geekbrains

## _Инструкция для **Git**_

git --version - команда для проверки версии git

git init - инициализация локального репозитория

git status - получить информацию от git о его текущем состоянии

git add - добавить файл или файлы к следующему коммиту

git commit -m "massage" - создание коммита

git log - вывод на экран истории всех коммтьов с их хеш кодами

git checkout - переход от одного коммтьа к другому

git checkout master - вернуться к актуальному состоянию и продолжить работу

git diff - увидеть разницу между текущим файлом и закоммиченым файлом

git branch - посмотреть список веток

git branch <название ветки> - создание новой ветки

git checkout <название ветки> - переход на другую ветку

git branch -d <название ветки> - удаление ветки

git push - Отправление репозитория на удаленный репозиторий

git pull - Выкачивание актуальной версии репозитория с удаленного и сливание с локальной версией

git clone ссылка - клонирование репозитория с интернета

git reset - спользуется в основном для отмены изменений

git clean - используется для удаления мусора из рабочего каталога

git show - отображает объект в простом и человекопонятном виде
