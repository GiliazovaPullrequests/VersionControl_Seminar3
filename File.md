# Инструкция по работе с Git

## Начальная работа с системой контроля версий

 **Git** – это распределенная система управления версиями.

 **GitHub** – это платформа для размещения кода для контроля версий и совместной работы. Это позволяет вам и другим пользователем работать вместе над проектами из любого места.
 В основе GitHub лежит система управления версиями с открытым исходным кодом (VSC), называемая Git. Git отвечает за все, что связано с GitHub, что происходит локально на вашем компьютере.

## **Основные команды Git**

***git --version*** – команда для проверки версии

***git init*** – инициализируем пустой репозиторий

***git status*** – показывает текущее состояние файлов

***git add*** – добавляет содержимое файла

***git commit*** – команда фиксации изменений файл

***git log*** – показывает журналы фиксации

***git checkout*** – переключение ветвей или восстановление файлов рабочего состояния

***git diff*** – показывает изменения между коммитом и рабочим

***git branch*** – выводит список веток в репозиторий

***git branch (название_ветки)*** – создает новую ветку

***git branch -d (название_ветки*** – удаляет ветку

***git checkout (название_ветки)*** – переключение между ветками

***git log -graph*** – выводит список коммитов в виде красивого древа

***git merge*** *название_ветки* – сливает ветку ***(название_ветки)*** с текущей веткой

***git push*** – обновляет удаленный репозиторий с помощью любых коммитов, сделанных локально в ветке

***git pull*** – скачать все из текущего репозитория и сделать merge с нашей версией

***git clone*** – клонирует внешний репозиторий на наш ПК

## **Заголовок**

Чтобы создать заголовок, добавьте цифровые знаки (#) перед словом или фразой. Количество используемых вами цифровых знаков должно соответствовать уровню заголовка.

# Уровень заголовка 1
## Уровень заголовка 2
### Уровень заголовка 3
#### Уровень заголовка 4
##### Уровень заголовка 5
###### Уровень заголовка 6


## **Альтернативный синтаксис**

В качестве альтернативы, в строке под текстом добавьте любое количество символов === для уровня заголовка 1 или --- для уровня заголовка 2.

Уровень заголовка 1
===================

Уровень заголовка 2
-------------------

## **Разрыв строк**

Если ваше приложение Markdown поддерживает HTML, то можете создать разрыв строки или новую строку (<br>), завершите строку двумя или более пробелами, а затем введите return.

<p>Это первая строка.<br>
А это вторая строка.<p>

## **Размер изображения**

Для изображений синтаксис Markdown не позволяет указывать ширину и высоту изображений. Если вам нужно изменить размер изображения, а если ваш процессор Markdown поддерживает HTML, то вы можете использовать img HTML-тег с атрибутами width и height, чтобы задать размеры изображения в пикселях.

<img src="MountFuji.jpg" width="203,55" height="350">

## **Подписи к изображениям**

Если ваше приложение Markdown поддерживает HTML, то вы сможете использовать HTML-теги figure и figcaption для добавления подписи к изображению.

<figure>
    <img src="MountFuji.jpg"
         alt="MountFuji">
    <figcaption>A shot of Mount Fuji from the street in Japan – Awesome.</figcaption>
</figure>

## **Выделение шрифта**

## Bold

Выделите жирным шрифтом две звездочки или подчеркивания перед и после слова или фразы. Чтобы выделить середину слова жирным шрифтом для выделения, добавьте две звездочки без пробелов вокруг букв.

|Markdown                     |HTML                   |
|-----------------------------|-----------------------|
|**Текст выделенный жирным**  |<strong>bold</strong>  |
|__Текст выделенный жирным__  |<strong>bold</strong>  |

## Italic

Чтобы выделить текст курсивом, добавьте одну звездочку или подчеркивание до и после слова или фразы. Чтобы выделить середину слова курсивом для выделения, добавьте одну звездочку без пробелов вокруг букв.

|Markdown                      |HTML               |
|------------------------------|-------------------|
|*Текст выделенный курсивом*   |<em>Italic</em>    |
|_Текст выделенный курсивом_   |<em>Italic</em>    |

## Bold and Italic

Чтобы выделить текст жирным шрифтом и курсивом одновременно, добавьте три звездочки или подчеркивания до и после слова или фразы. Чтобы выделить середину слова жирным шрифтом и курсивом для выделения, добавьте три звездочки без пробелов вокруг букв.

|Markdown                                | HTML                                     |
|----------------------------------------|------------------------------------------|
|***Текст выделенный жирным и курсивом***|<em><strong>Bold and Italic</strong></em> |
|__Текст выделенный жирным и курсивом__  |<em><strong>Bold and Italic</strong></em> |

## **Списки**

Вы можете упорядочивать элементы в нумерованные и ненумерованные списки.

## Нумерованные списки
Чтобы создать упорядоченный список, добавьте позиции с номерами, за которыми следуют точки. Числа не обязательно должны быть в числовом порядке, но список должен начинаться с цифры один.
1. Первый пункт
2. Второй пункт
3. Третий пункт

## Ненумерованные списки
Чтобы создать неупорядоченный список, добавьте тире (-), звездочки (*) или плюс (+) перед позициями. Сделайте отступ для одного или нескольких элементов, чтобы создать вложенный список.

- Первый пункт
+ Второй пункт
* Третий пункт

## **Форматирование таблицы**

Вы не можете использовать много элементов синтаксиса Markdown для форматирования текста в ячейках таблицы. Но есть обходные пути, по крайней мере, для двух распространенных проблем с таблицами: разрывы строк и списки.

## Разрывы строк в ячейках таблицы
Вы можете разделять абзацы внутри ячейки таблицы с помощью одного или нескольких HTML-тегов <br>

| Синтаксис   | Описание    |
| ----------- | ----------- |
| Заголовок   | Название |
| Абзац       | Первый абзац. <br><br> Второй абзац. |

## Списки в ячейках таблицы
Вы можете добавить список в ячейку таблицы с помощью HTML-тегов.

| Синтаксис   | Описание    |
| ----------- | ----------- |
| Заголовок   | Название |
| Абзац       | Вот список! <ul><li>Пункт первый.</li><li>Пункт второй.</li></ul> |

## **Автоматическая ссылка**

Многие процессоры Markdown автоматически превращают URL-адреса в ссылки. Это означает, что если вы наберете http://www.example.com , ваш процессор Markdown автоматически превратит его в ссылку, даже если вы не использовали скобки.

https://www.markdownguide.org/extended-syntax

## **Отключение автоматической ссылки**

Если вы не хотите, чтобы URL-адрес автоматически связывался, вы можете удалить ссылку, обозначив (`) до и после URL-адрес.

`https://www.markdownguide.org/extended-syntax/`

## **Идентификаторы заголовок**

Многие процессоры Markdown поддерживают пользовательские идентификаторы для заголовков — некоторые процессоры Markdown автоматически добавляют их. Добавление пользовательских идентификаторов позволяет вам напрямую ссылаться на заголовки и изменять их с помощью CSS. Чтобы добавить пользовательский идентификатор заголовка, заключите пользовательский идентификатор в фигурные скобки в той же строке, что и заголовок.

### Мой отличный заголовок {#custom-id}

## HTML-код выглядит следующим образом:

<h3 id="custom-id">Мой отличный заголовок</h3>
